## 优化记录

### 优化 1：增大快捷问题标签点击区域
- **问题：** 测试者反馈「拇指不好点」
- **改进方案：** 将 `.quick-tag` 的 padding 从 `8rpx 16rpx` 增至 `12rpx 24rpx`，高度确保不低于 60rpx。
- **改进后效果：** 点击区域增大约 40%，拇指操作更舒适。

### 优化 2：支持键盘发送
- **问题：** 单手操作时右下角发送按钮够不着（尤其大屏手机）
- **改进方案：** input 组件的 `confirm-type="send"` + `bindconfirm` 事件，用户点击键盘「发送」键即可发送消息。
- **改进后效果：** 无需点击发送按钮，键盘直接发送。

### 优化 3：优化 AI 回复等待体验
- **问题：** 用户不知道 AI 在处理，以为卡死
- **改进方案：**
  1. 发送后立即显示 typing 动画（三个跳动灰点）
  2. 在气泡中增加状态提示文字轮换：「正在查找知识库…」→「正在分析问题…」→「生成回答中…」
- **改进后效果：** 用户明确感知 AI 正在处理，等待容忍度提升。

## 未采纳建议及原因

| 建议 | 原因 |
|------|------|
| 深色模式 | 开发工作量大，基础版本优先保证日间体验；可在后续版本添加 |
| AI 回答折叠 | 目前回答通常 3-6 步，不会太长；过长回答优先优化提示词缩短，而非前端折叠 |
| 自定义快捷问题 | 涉及用户偏好存储和个性化推荐系统，超出基础版本范围 |
